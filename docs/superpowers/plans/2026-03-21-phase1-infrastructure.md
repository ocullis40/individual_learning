# Phase 1: Infrastructure Setup — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Set up the foundational infrastructure — Next.js app, PostgreSQL database, Prisma ORM, Vitest, and Stride integration — so that Phase 2 can be built entirely via Stride-orchestrated subagents.

**Architecture:** Next.js 15 App Router with TypeScript and Tailwind CSS. PostgreSQL via Prisma ORM. Vitest for testing. Stride plugin for task orchestration with project-specific hooks.

**Tech Stack:** Next.js 15, TypeScript, Tailwind CSS, PostgreSQL, Prisma, Vitest, Stride (Claude Code plugin)

### Task 3: Set Up Prisma and Database Connection

**Files:**

- Create: `.env` (gitignored — contains DATABASE_URL)
- Modify: `.gitignore` — add `.env`

- [ ] **Step 1: Install Prisma**

```bash
npm install prisma --save-dev
npm install @prisma/client
```

- [ ] **Step 2: Initialize Prisma with PostgreSQL**

```bash
npx prisma init --datasource-provider postgresql
```

This creates `prisma/schema.prisma` and a `.env` file. We will use `.env` instead (Next.js convention).

- [ ] **Step 3: Update .env with the correct DATABASE_URL**

`npx prisma init` generates a `.env` with a placeholder `DATABASE_URL`. Update it with your actual local database URL:
```
DATABASE_URL="postgresql://$(whoami)@localhost:5432/individual_learning_dev"
```

Note: Replace `$(whoami)` with your actual macOS username when creating the file. No password needed for local PostgreSQL default config.

- [ ] **Step 4: Verify .env is in .gitignore**

Check that `.gitignore` includes `.env` or `.env*`. The `create-next-app` generated `.gitignore` includes `.env*.local`. Add `.env` explicitly if it is not covered:

```
.env
```

- [ ] **Step 5: Write the Prisma schema with the User table**

Replace `prisma/schema.prisma` with:

```prisma
generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}

model User {
  id        String   @id @default(cuid())
  email     String   @unique
  name      String
  createdAt DateTime @default(now()) @map("created_at")

  @@map("users")
}
```

Note: Only the User table for Phase 1. Remaining tables (Topic, Lesson, etc.) are added in Phase 2 via Stride tasks.

- [ ] **Step 6: Run the initial migration**

```bash
npx prisma migrate dev --name init
```

Expected: Migration created and applied. `prisma/migrations/` directory created with SQL file.

- [ ] **Step 7: Create the Prisma client singleton**

Create `src/lib/prisma.ts`:

```typescript
import { PrismaClient } from "@prisma/client";

const globalForPrisma = globalThis as unknown as {
  prisma: PrismaClient | undefined;
};

export const prisma = globalForPrisma.prisma ?? new PrismaClient();

if (process.env.NODE_ENV !== "production") {
  globalForPrisma.prisma = prisma;
}
```

This prevents multiple Prisma clients during Next.js hot reload in development.

- [ ] **Step 8: Verify the connection works**

```bash
npx prisma migrate status
```

Expected: Output shows the `init` migration has been applied. No pending migrations.

Optionally, open Prisma Studio to visually inspect:

```bash
npx prisma studio
```

Expected: Browser opens showing the `User` model with `id`, `email`, `name`, `createdAt` columns.

- [ ] **Step 9: Commit**

```bash
git add prisma/schema.prisma prisma/migrations src/lib/prisma.ts .gitignore
git commit -m "feat: set up Prisma with PostgreSQL and User table"
```

Note: Do NOT add `.env` — it contains local database credentials.

### Task 4: Set Up Vitest

**Files:**
- Create: `vitest.config.ts`
- Create: `tests/lib/prisma.test.ts`
- Modify: `package.json` — add test script
- Modify: `tsconfig.json` — include tests directory

- [ ] **Step 1: Install Vitest and dependencies**

```bash
npm install vitest @vitejs/plugin-react --save-dev
```

- [ ] **Step 2: Create Vitest configuration**

Create `vitest.config.ts`:

```typescript
import { defineConfig } from "vitest/config";
import path from "path";

export default defineConfig({
  test: {
    globals: true,
    environment: "node",
    include: ["tests/**/*.test.ts", "tests/**/*.test.tsx"],
  },
  resolve: {
    alias: {
      "@": path.resolve(__dirname, "./src"),
    },
  },
});
```

- [ ] **Step 3: Add test script to package.json**

Add to the `"scripts"` section of `package.json`:

```json
"test": "vitest run",
"test:watch": "vitest"
```

- [ ] **Step 4: Update tsconfig.json to include the tests directory**

In `tsconfig.json`, add `"tests"` to the `"include"` array so TypeScript recognizes test files:

```json
"include": ["next-env.d.ts", "**/*.ts", "**/*.tsx", ".next/types/**/*.ts", "tests/**/*.ts"]
```

Note: The exact `include` array may vary depending on what `create-next-app` generated. The key addition is `"tests/**/*.ts"`.

- [ ] **Step 5: Write a smoke test for the Prisma connection**

Create `tests/lib/prisma.test.ts`:

```typescript
import { describe, it, expect } from "vitest";
import { prisma } from "@/lib/prisma";

describe("Prisma client", () => {
  it("connects to the database", async () => {
    const result = await prisma.$queryRaw<[{ result: number }]>`SELECT 1 as result`;
    expect(result[0].result).toBe(1);
  });

  it("can query the users table", async () => {
    const users = await prisma.user.findMany();
    expect(Array.isArray(users)).toBe(true);
  });
});
```

- [ ] **Step 6: Run the tests**

```bash
npm test
```

Expected: 2 tests pass. Vitest resolves the `@/` alias via `vitest.config.ts` and connects to the database using `DATABASE_URL` from `.env`.

- [ ] **Step 7: Commit**

```bash
git add vitest.config.ts tests/lib/prisma.test.ts package.json
git commit -m "feat: set up Vitest with Prisma connection smoke tests"
```
