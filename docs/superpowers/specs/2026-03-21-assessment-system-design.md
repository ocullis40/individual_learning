# Assessment System — Design Spec

## Purpose

Add two assessment mechanisms to the learning platform: conversational assessment within the lesson chat, and a per-topic short-answer quiz. Both are AI-powered, update the learner's TopicMastery, and provide actionable feedback.

## Two Assessment Types

### 1. Conversational Assessment (Chat-Based)

Triggered by the user saying "quiz me" (or similar) during a lesson chat conversation. Claude shifts from tutoring mode to assessment mode:

1. Asks a comprehension question about the current lesson
2. User answers in their own words
3. Claude evaluates the answer
4. If correct — acknowledges and asks another question (or returns to tutoring)
5. If incorrect/incomplete — teaches the concept right there, then re-asks to confirm understanding
6. Updates TopicMastery based on demonstrated knowledge

**Key behaviors:**
- User-initiated only (never interrupts unprompted)
- Questions are conversational, not formal (feels like a tutor asking, not an exam)
- Claude teaches on the spot when gaps are found (don't just say "wrong")
- TopicMastery updated based on performance
- Uses the existing chat infrastructure — no new API routes needed, just prompt changes

### 2. Short-Answer Topic Quiz

A formal 10-question quiz per topic, taken sparingly. Questions generated by Claude at runtime.

**Flow:**
1. User navigates to a topic and clicks "Take Quiz"
2. App calls Claude to generate 10 short-answer questions based on all lessons in that topic
3. Questions presented one at a time, user types answers
4. After all 10, user submits
5. All answers sent to Claude (Haiku) for grading in one call
6. Results shown: score, per-question feedback, suggested review sections
7. TopicMastery updated based on score

**Key behaviors:**
- Questions generated fresh each attempt (never memorizable)
- One question at a time, no peeking at feedback until all submitted
- Grading uses Haiku for cost efficiency
- Feedback includes: correct/incorrect, what was missed, which lesson section to review
- Score thresholds update TopicMastery (e.g., 80%+ = proficient, 50-79% = familiar, <50% = learning)

## Prompt Caching

Enable Anthropic prompt caching for lesson content in the chat system prompt. The lesson content (~4,000 tokens) is the same across all messages in a conversation — caching reduces input cost by ~90% after the first message.

## Data Model Changes

### TopicMastery (new)
- id, user_id, topic_id (FK), mastery_level (enum: not_started, learning, familiar, proficient), last_assessed_at, updated_at
- Unique constraint on user_id + topic_id

### QuizAttempt (new)
- id, user_id, topic_id (FK), questions (JSON — the generated questions), answers (JSON — user's answers), score (Int), feedback (JSON — per-question feedback from grading), created_at

Note: No Quiz table — questions are generated at runtime and stored with the attempt.

## API Routes

| Method | Route | Description |
|--------|-------|-------------|
| POST | `/api/topics/[id]/quiz/generate` | Generate 10 short-answer questions for a topic |
| POST | `/api/topics/[id]/quiz/grade` | Grade submitted answers, return feedback, update mastery |
| GET | `/api/profile/mastery` | Get all TopicMastery records for current user |

## Chat Prompt Changes

Update the chat system prompt to include assessment mode instructions:

```
...existing tutor instructions...

ASSESSMENT MODE:
If the user says "quiz me", "test me", "check my knowledge", or similar:
- Switch to assessment mode
- Ask a comprehension question about the lesson content
- When the user answers:
  - If correct: acknowledge briefly, optionally ask another question
  - If incorrect or incomplete: explain the correct concept clearly, then ask a follow-up to confirm understanding
- After assessment, you may update the user on how well they demonstrated understanding
- Return to normal tutoring mode when the user is done being quizzed
```

| Grade quiz | Haiku | ~3,000 tokens | ~2,000 tokens | ~$0.003 |

## Tech Details

### Prompt Caching Implementation
Use Anthropic's cache control headers on the system prompt:
```typescript
system: [
  {
    type: "text",
    text: systemPromptTemplate,
    cache_control: { type: "ephemeral" }
  },
  {
    type: "text",
    text: `LESSON CONTENT:\n${lessonContent}`,
    cache_control: { type: "ephemeral" }
  }
]
```

### Quiz Generation Prompt
```
Generate 10 short-answer questions to assess understanding of the following topic and its lessons.
Questions should test comprehension and application, not just recall.
Each question should be answerable in 1-3 sentences.

Return as JSON array:
[{ "question": "...", "referenceAnswer": "...", "lessonTitle": "..." }]

TOPIC: {topic.name}
LESSONS:
{all lesson content for this topic}
```

### Quiz Grading Prompt (Haiku)
```
Grade these short-answer responses. For each question, determine if the answer demonstrates understanding.

Return JSON array:
[{ "questionIndex": 0, "correct": true/false, "feedback": "...", "suggestedReview": "..." }]

Questions and answers:
{questions with user answers}
```

## Mastery Level Thresholds

| Score | Mastery Level |
|-------|--------------|
| 80-100% | proficient |
| 50-79% | familiar |
| 20-49% | learning |
| 0-19% | not_started (needs to revisit) |

## Out of Scope
- Multiple choice questions
- Timed quizzes
- Quiz history / retake comparisons
- Adaptive question difficulty
